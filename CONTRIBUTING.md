# Contributing

Thank you for your interest in this project!

## Project Status

This is a **portfolio/demonstration project** using synthetic CMS DE-SynPUF data to showcase healthcare analytics best practices.

**Not accepting code contributions** at this time, as this is a personal portfolio piece. However, feedback and questions are welcome!

---

## How to Provide Feedback

### Report Issues
If you find a bug or have a suggestion:
1. Open an [Issue](../../issues)
2. Use a clear, descriptive title
3. Include steps to reproduce (if applicable)
4. Tag appropriately: `bug`, `enhancement`, `question`, etc.

### Ask Questions
Questions about implementing this approach with real healthcare data:
- Open an Issue with the `question` label
- Describe your use case and specific challenge
- I'll respond when I can!

### Share Your Implementation
If you've adapted this pattern for your own work:
- Share your experience in [Discussions](../../discussions) (if enabled)
- Connect on LinkedIn (see profile in repository description)

## Development Setup

If you want to run this project locally for learning:
1. See [docs/REPRO_STEPS.md](docs/REPRO_STEPS.md) for full setup guide
2. Download CMS DE-SynPUF data (see [docs/DATA_POLICY.md](docs/DATA_POLICY.md))
3. Configure BigQuery and dbt (see [docs/CONNECTION_NOTES.md](docs/CONNECTION_NOTES.md))
4. Run `dbt run --select marts` to build models

**Estimated time:** 2-4 hours first-time setup
